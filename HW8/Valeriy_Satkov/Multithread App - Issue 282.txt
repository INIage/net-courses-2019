Multithread App #282

Нужно написать приложение которое будет искать ссылки на страницы в википедии и добавлять их в базу данных.

На вход приложению дается любой адрес из википедии:
Например вот такой: https://en.wikipedia.org/wiki/The_Mummy_(1999_film).

Часть 1. Один поток:

    Скачиваем страницу на жесткий диск.
    Мы ищем на странице html теги <a href... и извлекаем из нее атрибут url.
    Записываем найденные ссылки в базу данных. Ссылки не должны повторяться в таблице.

Часть 2. Многопоточность:

    По найденным ссылкам в части 1 организуем максимальное количество потоков.
    Внутри каждого потока выполняем действия 2 и 3 (часть 1).
    Повторяем рекурсивно до тех пор пока приложение не завершиться.

В базе данных должна быть одна таблица:
dbo.Links с двумя полями: Links, IterationId (для того, чтобы прочитать ссылки из предыдущей итерации).

Приложение должно не падать, если я запущу их несколько с различными стартовыми ссылками.

Юнит тесты которые должны быть и работать без интернета:
ShouldDownloadPage
ShouldExtractHtmlTags (ресурный файл - пример страницы необходим)
ShouldSaveTagsIntoDatabase
ShouldCallParsingForEachPageFromPreviousIteration

Ссылки которые сохраняются в базу должны быть только в пределах хоста стартовой страницы т.е. https://en.wikipedia.org

Нужно с помощью Thread.Sleep или Task.Wait избежать блокировки со стороны сервера - владельца страницы.
